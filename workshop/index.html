CONTEXT

Claude Code 不是 ChatGPT

同樣是 AI 對話，但能做的事差非常多。

純對話
ChatGPT · Claude.ai

你問，它答。

產出的是「文字」— 你要自己複製貼上到 Figma、Notion、Docs，再手動串起來。

有手有腳
Claude Code

你說，它做。

能讀你的訪談稿、寫出 prototype HTML、開瀏覽器秀給你看、在 Figma 建 frame、在 Notion 寫頁面。

對 UX 的意義 不再「複製貼上搬資料」— AI 直接從你的工作環境讀寫。

ROUND 01 · 一頁式

快速跑完四階段

INTERLUDE

Round 2 之前
先認識四個工具

CLAUDE.md　·　Skills　·　MCP　·　Plan Mode + Hooks

CONCEPT 01 / 04

CLAUDE.md

放在專案資料夾的「給 AI 看的 brief」— 每次新對話自動讀。

CLAUDE.md
# 產品 產品名稱: 咖啡店 App 目標用戶: 25-40 上班族 # Persona 講求速度、不喝甜的 # 設計原則 - 不要 emoji - 不要漸層 - 品牌色 #c2410c
放什麼

內容

  • 產品是什麼、賣給誰
  • Persona、痛點、設計原則
  • 研究發現、設計系統、品牌色
  • 你不要 AI 做什麼
類比

像 ChatGPT 的「自訂指令」 — 但範圍只在「這個專案」。寫一次，之後每次對話自動帶入。

CONCEPT 02 / 04

Skills

別人（或你自己）寫好的「專業流程 SOP 套件」，AI 自動在對的時機呼叫。

CLAUDE.md
WHAT

這個專案的「事實」

例：產品定位、persona、品牌色、設計原則

Skill
HOW

處理事情的「流程」，跨專案都能用

例：跑 thematic analysis、做 design review 的順序

UX 場景特別有用

內建 skills

brainstorming　發想前釐清

design-shotgun　一次 4 個變體

design-html　設計變真網頁

design-review　設計挑錯

自己寫

你的研究 SOP，也能包成 Skill

研究 SOP、評估 checklist、設計 review 流程，都能寫成你自己的 skill。

用 writing-skills 引導你寫 — 是「寫 skill 的 skill」。

PART 02 · 50 MIN

Round 2
完整跑法

有 brief、有 SOP、有專業評審 — 看 AI 變得多不一樣

PREP

準備兩個檔案

兩個都放到「你的工作資料夾」— 就是你之後打 claude 指令時所在的那個資料夾。

①
下載 Skills .claude/ 資料夾 下載後整個資料夾複製到你的工作資料夾，Claude Code 會自動載入。
→ Google Drive
②
CLAUDE.md 範本 填好範本，存成 CLAUDE.md 放在你的工作資料夾。 
# 我的產品
產品名稱：[填]　　一句話描述：[填]　　目標用戶：[填]

# 我的研究風格
研究方法偏好：[訪談 / 問卷 / 觀察 / 混合]
分析時要特別關注：[情緒？行為？文化脈絡？]

# 我的設計風格
視覺調性：[極簡 / 活潑 / 專業 / 溫暖...]
品牌色：[#色號]　　不要出現的元素：[例：emoji、漸層、卡通]

# 我的評估重點
最在意：[效率？學習曲線？情感體驗？]
PHASE 01 · 完整版

研究計劃

Research Plan — 用 brainstorming skill 釐清，再依 CLAUDE.md 產出。

PROMPT
用 brainstorming skill 先幫我釐清研究範圍。然後依照 CLAUDE.md 裡的研究方法偏好，產出完整研究計劃：研究目標、招募條件、訪談題綱（10 題，要包含我關注的維度）、分析計劃、時程。
對比 Round 1 有沒有把你的研究風格體現出來？題綱深度差在哪？
PHASE 02 · 完整版

研究報告

Synthesis — 依 CLAUDE.md 關注維度挖深，產出 persona + journey。

PROMPT
這是 5 個訪談逐字稿（附檔）。請：①做 thematic analysis，產出 3-5 個主題　②依照 CLAUDE.md 我關注的維度，特別挖深　③產出 2 個 persona、user journey、機會點清單。
對比 Round 1 AI 找到的 theme 跟你手動編 code 比，哪個更深？有沒有找到你沒想到的？
PHASE 03 · 完整版

Wireframe & Prototype

設計變體探索 → 高保真 prototype，符合 CLAUDE.md 視覺調性。

PROMPT
用 design-shotgun 根據前面的 persona + journey，發想 4 個首頁設計方向。我選一個之後，用 design-html 做成高保真 prototype，依照 CLAUDE.md 的視覺調性和品牌色。完成後幫我開瀏覽器。
對比 Round 1 有沒有真的呼應你的設計風格？視覺品質差多少？
PHASE 04 · 完整版

Evaluation

Usability critique — 雙重檢查：Nielsen heuristics + design-review skill。

PROMPT
請對 Phase 3 的 prototype 做雙重檢查：①用 Nielsen 10 heuristics 評估，每條給：問題、嚴重度（1-4）、改善建議　②用 design-review 做視覺一致性檢查　③特別針對 CLAUDE.md 的「最在意」維度深入評估。
對比 Round 1 抓到的問題深度差多少？跟你實際 usability test 找到的有重疊嗎？
REFLECTION

兩輪跑完，回頭看

關鍵觀察
關鍵不在 AI 寫得多好 — 在你怎麼配置 AI。

  • 產出有多通用？
  • 感覺像「AI 寫的」嗎？
ROUND 02 · 完整版

配置好的 AI

  • 哪一階段差異最大？
  • 哪個 component 貢獻最多？
  • 你的判斷在哪幾步無可取代？
TAKEAWAYS

帶回去繼續玩

  • 挑一個今天沒用到的 skill 試試
  • 拿你以前做完的研究報告（Phase 02），請 AI 蒸餾成可重用的 skill
  • 把你的研究 SOP 包成 skill（用 writing-skills）
  • 接一個 MCP 到 Figma 或 Notion
  • 設一個 PostToolUse hook 記錄你的設計迭代
AI 不會讓你變成 UX 設計師。
但會 放大 你已經會的事。

判斷、品味、對使用者的同理 — 這些才是核心。
