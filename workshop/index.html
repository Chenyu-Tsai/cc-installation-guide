CONTEXT

Claude Code 不是 ChatGPT

同樣是 AI 對話，但能做的事差非常多。

純對話
ChatGPT · Claude.ai

你問，它答。

產出的是「文字」— 你要自己複製貼上到 Figma、Notion、Docs，再手動串起來。

有手有腳
Claude Code

你說，它做。

能讀你的訪談稿、寫出 prototype HTML、開瀏覽器秀給你看、在 Figma 建 frame、在 Notion 寫頁面。

對 UX 的意義 不再「複製貼上搬資料」— AI 直接從你的工作環境讀寫。

ROUND 01 · 一頁式

快速跑完四階段

每張卡複製 prompt，貼進同一個 Claude Code session（AI 才記得上一步）。每個跑 5-7 分鐘就往下。

01
研究計劃

幫我做一份 [產品] 的 UX 研究計劃。

02
研究素材

根據我們的主題，製作所有可以設計 prototype 的素材。

03
Prototype

根據產生的素材，幫我做一個 HTML-based 的 prototype。

⚠️ 跑之前先跟 CC 確認 scope，不然會生到吐。

04
Evaluation

幫我評估這個 prototype 的可用性問題。

05
Final

根據評估結果，生成一個 final 的 prototype，命名為 final.html。

PAUSE

Round 1 跑完，先停一下

不討論好不好。先記下這三件事，等下 Round 2 跑完要回頭比對。

01

速度

02

落差

03

驚喜

有沒有意外讓你眼睛一亮？AI 看到你沒看到的？

INTERLUDE

Round 2 之前
先認識五個工具

CLAUDE.md　·　Skills　·　MCP　·　Plan Mode + Hooks　·　Session

CONCEPT 01 / 05

CLAUDE.md

放在專案資料夾的「給 AI 看的 brief」— 每次新對話自動讀。

CLAUDE.md
# 產品 產品名稱: 咖啡店 App 目標用戶: 25-40 上班族 # Persona 講求速度、不喝甜的 # 設計原則 - 不要 emoji - 不要漸層 - 品牌色 #c2410c
放什麼

內容

  • 產品是什麼、賣給誰
  • Persona、痛點、設計原則
  • 研究發現、設計系統、品牌色
  • 你不要 AI 做什麼
類比

像 ChatGPT 的「自訂指令」 — 但範圍只在「這個專案」。寫一次，之後每次對話自動帶入。

CONCEPT 02 / 05

Skills

別人（或你自己）寫好的「專業流程 SOP 套件」，AI 自動在對的時機呼叫。

CLAUDE.md
WHAT

這個專案的「事實」

例：產品定位、persona、品牌色、設計原則

Skill
HOW

處理事情的「流程」，跨專案都能用

例：跑 thematic analysis、做 design review 的順序

UX 場景特別有用

內建 skills

brainstorming　發想前釐清

design-shotgun　一次 4 個變體

design-html　設計變真網頁

design-review　設計挑錯

自己寫

你的研究 SOP，也能包成 Skill

研究 SOP、評估 checklist、設計 review 流程，都能寫成你自己的 skill。

用 writing-skills 引導你寫 — 是「寫 skill 的 skill」。

CONCEPT 05 / 05

Session

你跟 AI 的一次對話 = 一個 session。下面 6 個指令決定你怎麼開、看、收、關。

claude
開新 session
從 terminal 啟動 Claude Code，從零開始一段對話。
claude -r
接續上一次
列出歷史 session 給你挑一個接續。Desktop 版可以從 sidebar 看到。
/context
看用量
這個 session 目前用了多少 token、context 還剩多少。
/compact
濃縮對話
把長對話壓縮成摘要，騰出 context 繼續跑。
/clear
清空重來
同一個視窗開新 session，AI 忘掉前面所有對話（設定保留）。
/exit
離開
關掉 Claude Code 回到 terminal。session 紀錄會保留。

在 Round 1 的 CC 裡先 /exit　→　跑下面這行　→　手打 claude 開新 session。

cd .. && cd full && pwd
②

下載 Skills 放進 full/

點下面連結 → 到 Google Drive 把整個 .claude/ 資料夾下載下來 → 放進 full/，Claude Code 會自動載入。

開啟 Google Drive 下載 .claude/ →
③

填 CLAUDE.md，存進 full/

複製範本，填好 4 行，存成 full/CLAUDE.md。

# 產品　名稱：[填]　一句話：[填]　目標：[填]
# 研究風格　方法：[訪談 / 問卷 / 觀察 / 混合]　關注：[情緒？行為？文化？]
# 設計風格　調性：[極簡 / 活潑 / 專業]　品牌色：[#]　避免：[emoji / 漸層]
# 評估重點　最在意：[效率 / 學習曲線 / 情感體驗]
SKILLS · 13 個

這次會載入的 skill

放進 full/.claude/skills/ 後，Claude Code 會在對的時機自動呼叫。

研究 · Research
  • brainstorming
    動工前釐清要做什麼。
  • research-plan
    寫出問題、假設、招募、時程。
  • interview-guide
    長出 10-12 題半結構訪綱。
  • interview-simulator
    用 persona 生擬真逐字稿。
  • thematic-analysis
    萃主題 + persona + journey。
設計 · Design
  • design-consultation
    對話建立視覺系統，寫成 DESIGN.md。
  • design-shotgun
    一次生 N 個變體並排比較。
  • design-html
    變體做成 production HTML。
  • frontend-design
    不像 AI 味的前端介面。
評估 · Evaluation
  • heuristic-evaluation
    Nielsen 10 條 + severity + fix。
  • design-review
    設計師眼挑視覺與 AI slop。
工具 · Utility
  • browse
    headless Chrome 截圖 HTML / 網址。
  • writing-skills
    寫 / 改 / 驗證 skill 的 skill。
提醒 開新 session 後用 /skills 確認這 13 個都被抓到。
PHASE 01 · 完整版

研究計劃

從「我想研究 X」到完整研究計劃，2 步搞定。

開始前：在 full/ 底下手打 claude 開一個新的 session，再用 /skills 確認 skills 都有被 CC 抓到。

①
用 /brainstorming 幫我釐清要研究什麼。
Claude 會反問你 5-6 題（目標、constraints、假設）。回答到你覺得方向清楚為止。
②
根據剛剛的討論 + CLAUDE.md，用 /research-plan 產出完整計劃，存到 research/。
寫出 research/research-plan.md — 研究問題、objectives、hypotheses、招募條件、時程。
對比 Round 1 有沒有把你的研究風格體現出來？brainstorming 反問的題目有沒有逼出你沒想到的角度？
PHASE 02 · 完整版

研究一條龍

訪綱 → 模擬訪談 → 萃取 insight，3 個 skill 接力跑。

開始前：用 /clear 或 /compact 控管 context 量，避免越跑越笨。

①
看 research/research-plan.md，用 /interview-guide 寫訪綱，存到同一個 folder。
產出 research/interview-guide.md — 10-12 題、6 個 section、每題附 rationale + probes。
②
用 /interview-simulator 根據訪綱跑 5 個模擬訪談，放在 research/transcripts/。
Claude 會問你要哪 5 個 persona（或用 Adoption Curve 預設）。每份 3000+ 字。
③
用 /thematic-analysis 分析 research/transcripts/，產出 research/synthesis.md。
Open coding → axial coding → 3-5 themes + 2 personas + journey + 機會點清單。
對比 Round 1 AI 找到的 theme 跟你手動 code 比，哪個更深？模擬訪談跟真實訪談的落差在哪？
PHASE 03 · 完整版

設計三步

視覺系統 → 4 個變體 → 定稿，每一步都看得到截圖。

開始前：用 /clear 或 /compact 控管 context 量，避免越跑越笨。

①
根據 research/synthesis.md，用 /design-consultation 建立 DESIGN.md。
Claude 反問你視覺方向 5 題，提案 type / color / layout，截圖預覽給你看。
②
用 /design-shotgun 根據 persona + DESIGN.md，發想 4 個首頁變體，放在 design/variants/。
Claude 先提 4 個 concept 名稱讓你確認，再寫 4 個 HTML + 截圖 side-by-side 比較。
③
我選 v[X]，用 /design-html 變成 production 版 design/prototype.html。
寫 semantic HTML + 3 viewport（mobile / tablet / desktop）截圖驗證。
⚠️ 跑之前先跟 CC 確認 scope，不然會生到吐。
對比 Round 1 4 個變體有沒有真的 distinct？最後 prototype 跟你的視覺調性吻合多少？
PHASE 04 · 完整版

雙軌評估 + 修

結構化 + 設計師眼，兩條路徑互補，最後直接修嚴重的問題。

開始前：用 /clear 或 /compact 控管 context 量，避免越跑越笨。

①
用 /heuristic-evaluation 評 design/prototype.html。
截 3 viewport，每條 Nielsen heuristic 給 severity 1-4 + 證據 + 具體 fix。
②
用 /design-review 從設計師眼補一輪，存到 design/review.md。
抓 heuristic 抓不到的：視覺一致性、hierarchy、AI slop pattern。
③
對 severity ≥ 3 的問題直接修，修完 before / after 截圖比對。
Edit prototype.html → re-screenshot → Read PNG 驗證 → 重複到無 ≥ 3。
對比 Round 1 結構化（heuristic）跟設計師眼（design-review）抓到的問題重疊多少？修完問題的迭代速度差多少？
PHASE 05 · 完整版

產出 final prototype

把評估結果吃進去，做一個對齊 brand、避開 AI slop 的 final.html。

開始前：用 /clear 或 /compact 控管 context 量，避免越跑越笨。

①
整合 design/heuristic-eval.md + design/review.md，列出 final 版必改清單與保留清單。
輸出 design/final-spec.md — 之後 frontend-design 會吃這份當輸入。
②
用 /frontend-design 根據 design/final-spec.md + CLAUDE.md 重做 design/final.html。
嚴守 CLAUDE.md 的視覺調性與避免清單；不允許 AI slop（漸層、emoji 裝飾、generic font）。
對比 Round 1 Round 1 的 final.html 是 AI 自由發揮；Round 2 的 final 有 spec、有 brand、有 review 收斂 — 視覺差距有多大？

TAKEAWAYS

帶回去繼續玩

  • 挑一個今天沒用到的 skill 試試
  • 把你的研究 SOP 包成自己的 skill（用 writing-skills）
  • 想要更深更廣：Owl-Listener/designer-skills（91 個 UX skill，9 plugin）
  • 接 MCP 到 Figma 或 Notion，讓 AI 直接讀寫你的工作環境
  • 設 PostToolUse hook 自動記錄設計迭代
AI 不會讓你變成 UX 設計師。
但會 放大 你已經會的事。

判斷、品味、對使用者的同理 — 這些才是核心。
